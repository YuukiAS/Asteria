# 0.8.9 Clipboard Formula Result

## Summary

- Synced `main` from `origin/main` with a fast-forward pull to `ebc129d`.
- Reproduced the 0.8.9 real-browser issue in Microsoft Edge: copying a mixed editor selection with text, inline math, and block math produced clipboard HTML with math metadata, but pasting it back dropped the inline formula (`inlineMathCount=0`).
- Fixed paste normalization so Asteria removes rendered KaTeX clipboard children while preserving a lightweight LaTeX fallback inside math wrappers. This keeps ProseMirror's browser HTML parser from discarding empty inline math wrappers.
- Fixed the follow-up style loss shown in manual testing: if the clipboard stores equation color or highlight on a parent rich-text wrapper, Asteria now promotes that inherited style onto the math wrapper before paste parsing.
- Added same-app copy handling for selected math content so Asteria writes visible computed formula colors into clipboard HTML, then parses Asteria math clipboard HTML into styled math nodes on paste.
- Preserved inherited text marks while parsing Asteria math clipboard HTML so ordinary styled text copied beside equations keeps color, highlight, font size, and common inline marks.
- Fixed same-list paste insertion for copied Asteria list items so copying items 3 and 4 at the end of an ordered list appends clean items 5 and 6 instead of also injecting duplicate content into item 4.
- Fixed empty-current-line paste placement so copied list items replace the blank current list line instead of starting on the next line.
- Kept app version at `0.8.9`.
